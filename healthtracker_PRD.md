# PRD — Трекер здоровья (Health Tracker Bot)

**Версия:** 1.0  
**Дата:** 27 мая 2026  
**Статус:** Draft  
**Автор:** —

---

## 1. Обзор продукта

### 1.1 Описание

Health Tracker Bot — персональный ИИ-ассистент для мониторинга состояния здоровья. Приложение отслеживает ключевые физиологические показатели, ежемесячно напоминает о сдаче анализов, формирует персональное меню питания на основе результатов лабораторных исследований и контролирует соответствие показателей медицинским нормам.

### 1.2 Проблема

Большинство людей не отслеживают своё здоровье системно. Они забывают сдавать профилактические анализы, не знают, как питаться с учётом своих показателей, и замечают проблемы только когда они становятся серьёзными. Существующие решения либо слишком сложны (медицинские платформы), либо слишком упрощены (шагомеры и трекеры сна).

### 1.3 Решение

Единая панель, которая:
- агрегирует ключевые показатели (вес, давление, пульс, ИМТ);
- раз в месяц формирует персональный список анализов;
- на основе результатов анализов генерирует меню питания через ИИ;
- визуализирует динамику и отклонения от норм.

## 2. Цели и метрики

### 2.1 Бизнес-цели

- Повысить профилактическую осознанность пользователей.
- Снизить пропуск плановых анализов.
- Обеспечить персонализированные рекомендации по питанию без участия врача.

### 2.2 KPI

| Метрика | Цель (6 мес.) |
|---|---|
| DAU / MAU | ≥ 30% |
| Доля пользователей, сдавших анализы вовремя | ≥ 60% |
| Retention 30 дней | ≥ 40% |
| Открываемость ежемесячного напоминания | ≥ 70% |
| Средний NPS | ≥ 45 |

---

## 3. Целевая аудитория

### 3.1 Основной сегмент

**Взрослые 25–55 лет**, заботящиеся о здоровье превентивно. Имеют смартфон, базовую цифровую грамотность. Регулярно проходят профилактические осмотры или хотят начать.

### 3.2 Вторичный сегмент

Люди с хроническими заболеваниями (гипертония, диабет 2 типа, гипотиреоз), которым нужен ежедневный мониторинг показателей и строгий контроль питания.

### 3.3 Исключения

Дети до 18 лет. Пациенты в острых и неотложных состояниях (приложение не является медицинским изделием).

## 4. Функциональные требования

### 4.1 Модуль «Дашборд»

**Приоритет:** P0

| # | Требование |
|---|---|
| F-1.1 | Отображение карточек с текущими значениями: вес (кг), ИМТ, систолическое/диастолическое давление, пульс |
| F-1.2 | Цветовая индикация статуса каждого показателя: норма (зелёный), отклонение (жёлтый), критично (красный) |
| F-1.3 | График динамики за последние 7 дней (давление и пульс) |
| F-1.4 | Прогресс-бар прогресса к целевым значениям |
| F-1.5 | Блок ИИ-анализа: сводка состояния и рекомендации на основе всех текущих показателей |
| F-1.6 | Счётчик дней до следующей сдачи анализов |

### 4.2 Модуль «Ввод данных»

**Приоритет:** P0

| # | Требование |
|---|---|
| F-2.1 | Форма ввода физических показателей: вес, рост, систолическое давление, диастолическое давление, пульс, возраст, пол, уровень физической активности |
| F-2.2 | Форма ввода результатов анализов: глюкоза (ммоль/л), гемоглобин (г/л), общий холестерин (ммоль/л), ТТГ (мкМЕ/мл) |
| F-2.3 | Автоматический расчёт ИМТ после сохранения |
| F-2.4 | Журнал истории измерений с отображением последних записей |
| F-2.5 | Валидация введённых значений (диапазоны физиологических норм) |

### 4.3 Модуль «Анализы»

**Приоритет:** P0

| # | Требование |
|---|---|
| F-3.1 | Базовый ежемесячный список анализов: ОАК, глюкоза натощак, ОАМ |
| F-3.2 | Расширенный список раз в 3 месяца: липидный профиль, ТТГ |
| F-3.3 | Расширенный список раз в 6 месяцев: витамин D, витамин B12 |
| F-3.4 | ИИ-генерация персонального плана анализов на основе возраста, пола, текущих показателей и отклонений |
| F-3.5 | Отображение норм для каждого анализа |
| F-3.6 | Визуальный счётчик дней до следующей плановой сдачи |

### 4.4 Модуль «Питание»

**Приоритет:** P0

| # | Требование |
|---|---|
| F-4.1 | Базовое меню на день: завтрак, 2 перекуса, обед, ужин |
| F-4.2 | Отображение калорийности и БЖУ для каждого приёма пищи и итогово за день |
| F-4.3 | ИИ-генерация меню с учётом результатов анализов (глюкоза, холестерин, гемоглобин, ТТГ, давление, ИМТ) |
| F-4.4 | Обоснование выбора блюд: почему каждое полезно при текущих показателях |
| F-4.5 | Обновление меню по кнопке |
| F-4.6 | Прогресс-бар по калориям относительно суточной нормы |

### 4.5 Модуль «Настройки»

**Приоритет:** P1

| # | Требование |
|---|---|
| F-5.1 | Хранение личных данных: имя, дата рождения, целевой вес |
| F-5.2 | Настройка дня месяца для плановой сдачи анализов |
| F-5.3 | Управление напоминаниями: включить/выключить каждый тип отдельно |
| F-5.4 | Таблица медицинских норм для всех отслеживаемых показателей |
| F-5.5 | Поддержка тёмной и светлой темы (авто по системным настройкам) |

## 5. Нефункциональные требования

### 5.1 Производительность

- Время первой загрузки дашборда: ≤ 2 сек на 4G.
- Ответ ИИ-модуля (анализ, меню): ≤ 8 сек.
- Сохранение показателей: мгновенно (оптимистичный UI).

### 5.2 Доступность

- Поддержка WCAG 2.1 AA.
- Минимальный размер шрифта: 12px.
- Контраст текста: ≥ 4.5:1.
- Поддержка экранных дикторов (aria-label на всех иконках).

### 5.3 Безопасность и конфиденциальность

- Все данные хранятся локально на устройстве (localStorage / IndexedDB) — по умолчанию.
- Облачная синхронизация — опционально, с явным согласием пользователя.
- Передача данных в ИИ-модель — только агрегированные показатели без ПДн.
- Соответствие GDPR и законодательству РК об обработке персональных данных.

### 5.4 Совместимость

- Веб-версия: Chrome 110+, Safari 16+, Firefox 115+.
- Мобильная веб-версия: iOS 16+, Android 10+.
- Адаптивный дизайн: от 375px до 1440px.

## 6. ИИ-компоненты

### 6.1 Модель

Используется Claude claude-sonnet-4-20250514 через Anthropic API (`/v1/messages`).

### 6.2 Функции ИИ

| Функция | Вход | Выход |
|---|---|---|
| Анализ состояния | Все физические показатели + анализы | Текстовая сводка + рекомендации (3–5 предложений) |
| Персональный план анализов | Возраст, пол, отклонения в показателях | Структурированный список с обоснованием |
| Меню по анализам | Глюкоза, холестерин, гемоглобин, ТТГ, давление, ИМТ | 5 приёмов пищи с описанием и обоснованием |
| Генерация меню | Давление, ИМТ, холестерин | JSON с 5 блюдами (name, desc, kcal) |

### 6.3 Ограничения ИИ

- Приложение **не является медицинским изделием** и не ставит диагнозы.
- Все рекомендации носят информационный характер.
- При критических отклонениях система рекомендует обратиться к врачу.
- Токены на вызов: max_tokens = 1000.

## 7. UX и навигация

### 7.1 Структура навигации

```
Дашборд
├── Карточки показателей
├── График динамики
└── ИИ-анализ

Ввод данных
├── Физические показатели
├── Результаты анализов
└── История измерений

Анализы
├── Ежемесячный список
├── Персональный план (ИИ)
└── Счётчик дней

Питание
├── Меню на сегодня
├── БЖУ
└── Меню по анализам (ИИ)

Настройки
├── Личные данные
├── Напоминания
└── Таблица норм
```

### 7.2 Принципы UX

- Одна главная действие на экран.
- Статус показателя виден без взаимодействия (цвет + иконка).
- ИИ-кнопки помечены стрелкой ↗ и активируются явно.
- Напоминания управляются тогглами без перехода в системные настройки.

## 8. Нормы показателей (встроенные)

| Показатель | Норма |
|---|---|
| ИМТ | 18.5 – 24.9 |
| Систолическое давление | < 130 мм рт.ст. |
| Диастолическое давление | < 85 мм рт.ст. |
| Пульс в покое | 60 – 100 уд/мин |
| Глюкоза натощак | 3.9 – 5.5 ммоль/л |
| Общий холестерин | < 5.2 ммоль/л |
| Гемоглобин (мужчины) | 130 – 170 г/л |
| Гемоглобин (женщины) | 120 – 150 г/л |
| ТТГ | 0.4 – 4.0 мкМЕ/мл |

## 9. Ограничения и риски

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Неточные рекомендации ИИ | Средняя | Высокое | Дисклеймер «не является медицинским советом» |
| Пользователь вводит неверные данные | Высокая | Среднее | Валидация диапазонов + подсказки |
| Недоступность Anthropic API | Низкая | Среднее | Graceful fallback: статичные рекомендации |
| Утечка персональных данных | Низкая | Высокое | Хранение только на устройстве по умолчанию |
| Имитация медицинской консультации | Средняя | Высокое | Явный дисклеймер, рекомендация врача при критических значениях |

## 10. Roadmap

### v1.0 — MVP (текущая версия)
- Дашборд с 4 ключевыми метриками
- Ввод показателей и анализов
- Ежемесячный список анализов

## 11. Юридический дисклеймер

Приложение предназначено исключительно для информационных целей. Оно **не является медицинским изделием**, не ставит диагнозы и не заменяет консультацию врача. При наличии симптомов заболевания или критических отклонениях показателей необходимо обратиться к квалифицированному специалисту.
